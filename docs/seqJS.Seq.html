extract(feat, extract_features) → {seqJS.Seq}

Extract the sequence that the feature refers to TODO: find out which features should be kept
Parameters:
Name Type Argument Default Description
feat seqJS.Feature The feature whose underlying sequence we want to extract
extract_features boolean <optional>
 false Whether or not to include cropped features in the output seqJS.Seq. If 'feat' is a multi-span feature, then features which are present in multiple spans are kept as single features, with the spans merged using the 'order' merge operator. For example, if a seq has a feature whose location is 1..50, and you extract a feature whose location is join(10..19,30..39), the location of the resulting feature is order(1..10,11..20)
